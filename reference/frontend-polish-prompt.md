## 项目背景

项目名：SJTU HTML PPT Template

这是一个用于生成上海交通大学风格 HTML 幻灯片的个人学习项目。项目包含两类写法：

- 直接写 HTML / JavaScript。
- 使用类 Markdown 的 `.sjtu.md`，再转换为 HTML slide。

当前重点优化对象是本地可视化编辑器：

```text
editor/
  index.html
  editor.css
  editor.js
  server.js
```

编辑器当前定位是一个 IDE 式工作台：

- 左上：`.sjtu.md` 源文件编辑器。
- 左下：`.layout.json` 布局补丁编辑器。
- 右侧：生成后的 HTML slide 预览 iframe。
- 顶部工具栏：文件选择、打开、保存、构建、导入、导出、编辑/阅读模式、语言切换、动画和字号控制、插入片段按钮。

该编辑器需要服务于不熟悉 JavaScript 的用户，让他们能用 `.sjtu.md` 写内容，再通过可视化方式微调布局。

## 当前界面可能存在的问题

请假设当前编辑器已经能运行，但视觉层面仍偏粗糙：

- 顶部工具栏信息密度较高，分组不够优雅。
- 按钮、下拉框、输入框风格比较基础，缺少产品级质感。
- 左侧源码区和 layout 区只是普通 textarea，缺少“代码编辑器”的视觉层级。
- 右侧预览区和左侧编辑区的关系还不够像专业 IDE。
- 选择对象后的状态提示不够明显。
- 中英文切换已有基础功能，但语言选择控件和整体界面融合度还可以提升。
- 导入/导出、保存/构建等高频操作需要更清晰的主次关系。
- 移动端或窄屏下可用性需要考虑，但不必做成完整移动应用。

## 目标风格

整体风格应是：

- 现代、克制、专业的本地 IDE / slide editor。
- 面向学术报告和上海交通大学主题。
- 以白底、浅灰、红棕色系为主。
- 使用 SJTU 红作为强调色，但不要让页面变成单一红色主题。
- 有足够的呼吸感，但不能像营销 landing page。
- 更像“学术工具 / 排版工作台”，而不是 PPT 模板官网。

建议风格关键词：

- 清爽
- 稳重
- 工具感
- 低噪声
- 有层级
- 有明确焦点
- 与 SJTU slide 主题一致

避免：

- 大面积渐变背景。
- 装饰性光斑、圆球、漂浮图形。
- 过度圆角、过度阴影。
- 花哨卡片堆叠。
- 牺牲信息密度的营销式 hero 布局。
- 改动核心功能逻辑。

## 必须保留的功能

前端美化不能破坏以下功能：

- 加载 `.sjtu.md` 与 `.layout.json`。
- 文件下拉选择与打开。
- 保存并构建。
- 单独构建。
- 导入文件夹。
- 导出 ZIP。
- 编辑模式 / 阅读模式。
- 中文 / English 语言切换。
- 预览 iframe。
- 点击预览对象后定位源码和 layout。
- 拖拽、缩放预览对象并写回 layout。
- 设置 step、动画 effect、字号比例。
- 插入公式、定理、图片、分步片段。

## 设计细节建议

请在最终提示词中尽量具体到以下层面。

### 顶部工具栏

目标：从“很多控件堆在一起”变成“清晰的 IDE 命令栏”。

建议：

- 按功能分组：项目、构建、视图、对象属性、插入。
- 保存并构建作为主按钮。
- 打开、构建、导出作为次级按钮。
- 导入文件夹可以做成较轻的按钮或菜单入口。
- 当前文件路径用更弱的文字显示，避免抢占操作按钮焦点。
- 语言选择放在右侧或视图区域，不要干扰主要工作流。

### 左侧编辑区

目标：让 `.sjtu.md` 和 `.layout.json` 更像代码编辑器。

建议：

- 面板标题栏更清晰。
- 面板可以有轻微背景差异。
- textarea 使用更舒适的等宽字体、行高、padding。
- 聚焦状态明确但不要刺眼。
- 当前选中源码/layout 时可以有更明显的边框或标题强调。
- 两个编辑区高度比例保持合理。

### 右侧预览区

目标：让预览更像 slide canvas，而不是普通 iframe。

建议：

- 预览区域标题栏显示当前选择对象。
- iframe 周围留出稳定边距。
- 背景可以用浅灰工作台色，iframe 保持白底。
- 选择对象后的状态信息要清楚。
- 编辑模式与阅读模式有清晰但克制的视觉差异。

### 对象属性控制

目标：让 step、effect、fontScale 更像属性面板，而不是散落表单。

建议：

- 将属性控件视觉上归为“当前对象”。
- 没有选择对象时可显示弱提示，或禁用应用按钮。
- 控件标签中文更明确，例如“出现步数”“动画效果”“字号倍率”。

### 颜色与字体

建议：

- 主色：SJTU 红或红棕色。
- 辅色：暖灰、浅米白、深墨色。
- 危险或错误状态用红色，但与主色有层级差异。
- 成功状态可用低饱和绿色或文字提示，不要太跳。
- 字体以系统中文字体和等宽字体为主。

### 响应式

要求：

- 宽屏保持左侧双编辑区、右侧预览。
- 窄屏时上下堆叠，但工具栏不能完全失控。
- 工具栏允许换行，但按钮不应挤压到文字不可读。

### 可访问性

建议：

- 按钮和输入控件保留清晰 hover/focus 状态。
- iframe title 保留。
- 不要用纯颜色表达状态。
- 不要让文字与背景对比过低。
